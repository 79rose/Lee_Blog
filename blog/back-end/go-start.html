前言 ​

Golang是一门静态语言，同时他编译运行速度很快，具有良好的性能，同时它的语法十分有趣！非常值得学习！

快速开始 ​

首先初始化项目

在vsc上创建cards项目 然后打开终端 输入go mod init

然后依次创建 deck.go main.go deck_test.go文件

deck.go文件

每一个go文件 都得有它的归属包

包的作用就是相当于给文件分类 main包是编译运行的 而其他包只能编译

go
package main

import (
	"fmt"
	"math/rand"
	"os"
	"strings"
	"time"
)

type deck []string

func newDeck() deck {
	decksuits := []string{"Spades", "Hearts", "Diamonds", "Clubs"}
	deckvalue := []string{"Ace", "Two", "Three", "Four"}
	d := deck{}
	for _, suit := range decksuits {
		for _, value := range deckvalue {
			d = append(d, value+" of "+suit)
		}
	}
	return d
}

看这段代码，我们首先指定了它的归属包，然后引入了它的依赖包。 需要注意 在Vsc里，我们的import是自动执行的，它根据项目用到的包进行导入。

然后我们创建了一个 deck 类型 为什么我们要给[]string重新指定一个类型呢？ 因为go不是面向对象的语言，但是通过类型别名却可以实现面向对象的特性！

我们定义一个newdeck的函数，返回一个deck。在这里，我们用到了 := ，这是go的语法特性， 他可以根据你所指定的类型自动推断变量类型。

同时，在for循环里，条件是没有括号的，这个range作用于引用类型上， 此时suit变量会依次接收decksuits的值，通过循环给新创建的deck初始化。

创建好deck之后，我们可以给这个deck加一点功能

go
func (d deck) print() {
	for _, d := range d {
		fmt.Println(d)
	}
}

这个函数跟我们熟悉的函数大有不同，因为他在前面又接收了一个deck参数，这跟普通的函数又有什么区别呢？

通过接收这个参数，我们可以通过这个参数直接调用这个函数，这是不是我们熟悉的面向对象呢？

go
	d := newDeck()
        d.print()

这个参数我们称为接收器，可以理解为java里的this，python的self。 接收器参数是隐式调用的，我们也可以通过显示调用的方式来创建。

go
func  print(d deck) { for _, d := range d { fmt.Println(d) } }
print(newDeck())

接下来我们继续给这个deck增加一下功能

go
func (d deck) toString() string {
	return strings.Join([]string(d), ",")
}
func (d deck) saveToFile(fileName string) error {
	return os.WriteFile(fileName, []byte(d.toString()), 0666)
}

我们创建了一个saveToFile的方法，让deck的内容保存到当前目录下的fileName文件下（创建且复制）。在这个函数后面，我们返回了一个匿名error ，error也是一个类型，而os.WriteFile本身返回一个错误，所以我们直接把它返回。

注意os.WriteFile接收三个参数，其中第二个参数是[]byte类型，这是机器乐意识别的类型，我们想把deck从[]string 转成[]byte,首先我们新建一个tostring方法，把deck拼接成string，然后通过[]byte(string)就可以转成[]byte。

在main文件中，写入代码

go
package main

func main() {
	d := newDeck()
	d.saveToFile("my_deck")
	d.print()
}

运行命令

go run main.go deck.go

可以看到 my_deck文件被创建出来，同时控制台也打印了deck的信息

既然可以写文件，那我们还可以读一下文件！

go

func (d deck) newDeckOfFile(fileName string) deck {
	data, err := os.ReadFile(fileName)
	if err != nil {
		fmt.Println("error: ", err)
		os.Exit(1)
	}
	return strings.Split(string(data), ",")
}

在deck.go文件中，创建newDeckOfFile方法，读取文件并且返回一个deck。同样， os.ReadFile也会返回一个error，这次我们处理一下这个error。当读取文件成功，os.ReadFile不会返回error，这时返回的是一个nil，对于引用类型，nil就是null的意思，如果error不为空，我们就要打印错误信息，并且程序停止运行，立刻退出。如果没有错误，我们读取文件的字符串，然后拼成[]string返回。

既然是卡牌，我们怎么可以不洗牌

go
func (d deck) shuffle() {
	s := rand.NewSource(time.Now().UnixNano())
	r := rand.New(s)
	for i := range d {
		j := r.Intn(len(d))
		d[i], d[j] = d[j], d[i]
	}
}

我们使用rand函数进行随机洗牌，但是rand使用的随机数源是固定的，所以我们通过当前运行程序的时间，创建rand的种子，然后通过随机交换顺序完成洗牌。

go
package main

func main() {
	d := newDeck()
	d.saveToFile("my_deck")
	d.newDeckOfFile("my_deck")
	d.shuffle()
	d.print()
}

在main.go里，写入代码，运行一下，可以看到，每次运行的结果是不一样的。

代码测试 ​

完成了deck的一些功能，我们可以对他进行测试。

在deck_test.go文件里

go
package main
import "testing"
//引入测试
func TestNewDeck(T *testing.T) {
    deck := newDeck()
    if len(deck) != 16 {
        T.Errorf("Expected deck to have 16 cards, got %d", len(deck))
    }
}

testing 为 Go 语言 package 提供自动化测试的支持。通过 go test 命令，能够自动执行如下形式的任何函数：

func TestXxx(*testing.T)

我们创建TestNewDeck测试函数，然后进行一个简单的测试

go test

测试通过！
