web component ​

概念 ​

web component 允许你创建可重用的定制元素，由 Custom Element , Shadow Dom, HTML Template 组成

  • Custom Element 可以通过js类来继承原生内置元素 从而扩展元素功能
  • Shadow Dom 一个类似于scope 的概念 允许你将一个 DOM 树附加到一个元素上，并且使该树的内部对于在页面中运行的 JavaScript 和 CSS 是隐藏的 (一个独立的沙盒)
  • Html Template 可以通过js 动态渲染 可重用的dom结构

然后，我们实现我们的LazyImage 类

继承自 HTMLElement，这个时候可以监听组件对应的生命周期做出不同的操作； 我们选择监听点击和鼠标移入移除事件，调整对应的ui样式，让图片拥有变大效果 和对应的点击行为。

js

// 创建web component
class LazyImage extends HTMLElement {
    // 监听属性变化
    static get observedAttributes() {
        return ['src', ' class'];
    }

    constructor() {
        super();
        this.attachShadow({ mode: 'open' });
        this.shadowRoot.appendChild(template.content.cloneNode(true));
        this.img = this.shadowRoot.querySelector('img');
    }
    // 生命周期 当元素插入文档时调用
    connectedCallback() {
        this.addEventListener('click', this._handleClick.bind(this));
        this.addEventListener('mouseenter', this._handleMouseEnter.bind(this));
        this.addEventListener('mouseleave', this._handleMouseLeave.bind(this));
        this.observer = createObserver(this._loadImage.bind(this), this);
        this.observer.observe(this);
    }
    // 生命周期 当元素从文档中删除时调用
    onDisconnectedCallback() {
        this.removeEventListener('click', this._handleClick.bind(this));
        this.removeEventListener('mouseenter', this._handleMouseEnter.bind(this));
        this.removeEventListener('mouseleave', this._handleMouseLeave.bind(this));
        this.observer.disconnect();
    }
    // 生命周期 当元素的一个属性被增加、移除或更改时被调用
    attributeChangedCallback(name, oldValue, newValue) {
        console.log(name, oldValue, newValue);
    }
    get defaultPic() {
        return this.getAttribute('default-src') ?? 'https://picsum.photos/200/300';
    }

    get Src() {
        return this.getAttribute('src');
    }
    get getUrl() {
        return this.getAttribute('url');
    }
    get width() {
        return this.getAttribute('width');
    }
    get height() {
        return this.getAttribute('height');
    }
    _handleClick() {
        if (this.getUrl) {

            window.open(this.getUrl);
        }
    }
    _handleMouseEnter() {
        if (this.getUrl) {
            this.img.style.cursor = 'pointer';
        }
    }
    _handleError() {
        this.shadowRoot.querySelector('slot').style.display = 'block';
        this.img.style.display = 'none';
    }
    _handleMouseLeave() {
        this.img.style.cursor = 'default';
    }
    _loadImage() {
        const newImg = createImg(this.defaultPic, this.Src, this._handleError.bind(this));
        this.img.replaceWith(newImg);
        this.img.style.display = 'block';
        this._initSize();
    }
    _initSize() {
        this.img.style.width = this.width ?? '200px';
        this.img.style.height = this.height ?? '200px';
    }
}

核心作用：实现懒加载

我这边使用的思路是 利用IntersectionObserverAPI 监听图片是否处于视口，同时利用Image 的异步响应函数完成不同状态的响应，再将真实图片url赋值。

js
// 创建观察者  
const createObserver = (callback, target) => {
    const observer = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            console.log(entry);
            // 当元素进入视口时
            if (entry.isIntersecting) {
                callback();
                observer.unobserve(target);
            }

        })
    }
    )
    return observer;
}
// 图片懒加载
const createImg = (defaultPic, Pic, handleError) => {
    const img = new Image();
    img.src = defaultPic;
    img.onload = () => {
        setTimeout(() => {
            img.src = Pic;
        }, 1000);
    }
    img.onerror = () => {
        console.log('error');
        handleError();
    }
    return img;
}

于是我实现了vue版本的LazyImage component

vue
<template>
  <div class="wrap" @click="handleClick" ref="imgWrap">
    <img :src="img" ref="imgRef" class="img" :style="{
      width: `${width}px`,
      height: `${height}px`
    }" />
    <slot ref="slotRef" name="error">
      出错啦
    </slot>
  </div>
</template>

<script setup lang="ts">
import { ref, onMounted, defineProps, onUnmounted } from 'vue'

/**
 * @description: 懒加载图片组件 vue实现
 */
interface Props {
  src: string
  defaultSrc: string
  width: number
  height: number
  url: string
}

const { defaultSrc, width, height, url, src } = withDefaults(defineProps<Props>(), {
  width: 100,
  height: 100,
})

const imgRef = ref(null)
const img = ref(defaultSrc)
const slotRef = ref(null)
const loadImage = (src: string) => {
  return new Promise((resolve, reject) => {
    const image = new Image()
    image.onload = () => resolve(src)
    image.onerror = reject
    image.src = src
  })
}

const handleImageLoad = async (src: string) => {
  try {
    const loadedSrc = await loadImage(src)
    img.value = loadedSrc
  } catch {
    img.value = ''
    slotRef.value.style.display = 'block'
  }
}

const checkViewport = (callback: Function) => {
  const rect = imgRef.value.getBoundingClientRect()
  if (rect.top >= 0 && rect.bottom <= window.innerHeight) {
    setTimeout(() => {
      handleImageLoad(src)
    }, 1000)
  }
}

const handleClick = () => {
  if (url) {
    window.open(url)
  }
}

const init = () => {
  if (url) {
    imgRef.value.style.cursor = 'pointer'
    }
    const imge = imgRef.value
  if (imge) checkViewport()
}

onMounted(() => {
  init()
  window.addEventListener('scroll', checkViewport)
})
onUnmounted(() => {
  window.removeEventListener('scroll', checkViewport)
})
</script>

vue 与 web component 的联系 ​

相似之处

  • 组件化开发: 开发者可以将ui分离为可复用的模块
  • 复用性：可以通过组件库等形式 将组件复用到不同的项目

区别

  • 封装性： 毫无疑问，vue 的 sfc具有更 详尽的生命周期，开发者可以在不同的生命周期做更细致的操作
  • 状态管理：向内，vue有自己的状态管理机制(计算属性，响应式数据)，向外，vue有社区的pinia的全局状态管理库

vue 借鉴了 web component 的一些思路，在这个基础之上解决了一些webcomponent 本身的局限。

比如：

Vue的作用域插槽（插槽可以访问子组件的状态);

SSR 的 css scope 问题（vue本身提供了scope机制，就是动态在类名之后添加独特hash，实现局部css隔离，本质还是css优先级问题)...

Web Component的应用 ​

跨框架共享组件 （不需要引入第三方库,本身浏览器支持) | 组件库

比如流行的ui库material , 国内的 Taro(跨端，可以跑在小程序) ， 还有腾讯 的 omi-mp

微前端

首先说一下 微前端最基本要素：

  • 技术栈无关 （浏览器原生）
  • 应用隔离 （shadow）
  • 独立开发 （无需引入三方库）

其中 腾讯的wujie 就是基于web componnent 来实现的微前端

无界采用webcomponent来实现页面的样式隔离，将子应用的完整结构渲染在内部（shadow root）

子应用的实例在iframe内运行，dom在主应用容器下的webcomponent中，通过代理 iframe的document到webcomponent，可以实现两者的互联, 完成父子应用之间的通信创联。

其中web component 起到的作用就是 作为一个css沙箱（将主应用document的查询接口 quer代理到Web Component).
